React Native Expo Guide

Architecture and boilerplate for the FlowSplit hybrid financial engine. Built for performance, reliability, and clean aesthetics.

Folder Architecture

flowsplit-app/
├── src/
├── assets/ # Fonts, Images, SVGs
├── components/ # Atoms/Molecules (Button, Card, Chart)
├── constants/ # theme.ts, config.ts
├── hooks/ # useSplitCalculations, useDeposits
├── navigation/ # AppNavigator.tsx
├── screens/ # Main views
├── services/ # api.ts (Axios), storage.ts
├── types/ # TypeScript interfaces
└── utils/ # formatting.ts, validation.ts
├── App.tsx # Root entry
├── app.json # Expo configuration
└── package.json

PascalCase Components

Always name UI components using PascalCase (e.g., BucketCard.tsx).

camelCase Functions

Use camelCase for utilities and services (e.g., calculateRemainder.ts).

API Service (services/api.ts)

import axios from 'axios';

const api = axios.create({ baseURL: 'http://api.flowsplit.com/v1' });

export const fetchDeposits = () => api.get('/deposits');
export const createSplitPlan = (data) => api.post('/splits', data);
export const confirmSplit = (id) => api.patch(`/splits/${id}/confirm`);
